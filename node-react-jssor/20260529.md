# Pembetulan penapisan tarikh hebahan

Matlamat: tamat tempoh (`endDate`) menapis **secara automatik** tanpa `window.location.reload()` pada 12 AM.

Strategi: satu sumber logik (`isHebahanActive`) dikongsi, backend broadcast bila set aktif berubah (guna `hebahan:updated` yang sedia ada), dan frontend tapis pada masa-papar setiap 60 saat sebagai jaring keselamatan.

---

## 1. Backend — helper tarikh selamat waktu tempatan

**Fail baru: `nodejs/utils/hebahanDate.js`**

```js
/**
 * Logik tarikh hebahan (satu sumber kebenaran, dikongsi backend).
 * Parse waktu tempatan (elak isu UTC pada `new Date("YYYY-MM-DD")`).
 */

function parseLocalDate(str) {
  if (!str || typeof str !== 'string') return null;
  const m = str.trim().match(/^(\d{4})-(\d{2})-(\d{2})/);
  if (!m) return null;
  const d = new Date(Number(m[1]), Number(m[2]) - 1, Number(m[3]));
  return isNaN(d.getTime()) ? null : d;
}

/**
 * Tentukan sama ada hebahan aktif pada `now`.
 * - Tiada kedua tarikh  -> sentiasa aktif
 * - Mula sahaja          -> aktif bila hari ini >= mula
 * - Akhir sahaja         -> aktif sehingga hujung hari akhir (tamat tepat 00:00 keesokan)
 * - Kedua-dua ada        -> aktif jika dalam julat
 */
function isHebahanActive(startStr, endStr, now = new Date()) {
  const todayStart = new Date(now.getFullYear(), now.getMonth(), now.getDate(), 0, 0, 0, 0);
  const start = parseLocalDate(startStr);
  const end = parseLocalDate(endStr);
  const endOfEnd = end
    ? new Date(end.getFullYear(), end.getMonth(), end.getDate(), 23, 59, 59, 999)
    : null;

  if (!start && !endOfEnd) return true;
  if (start && !endOfEnd) return todayStart >= start;
  if (!start && endOfEnd) return now <= endOfEnd;
  return todayStart >= start && now <= endOfEnd;
}

module.exports = { parseLocalDate, isHebahanActive };
```

## 2. Backend — guna helper dalam `parseHebahan`

**Fail: `nodejs/services/dataService.js`** — ganti badan `parseHebahan` (kekalkan output yang sama supaya pemanggil sedia ada tak rosak).

```js
const { isHebahanActive } = require('../utils/hebahanDate');

// ...di dalam class...

parseHebahan(content, now = new Date()) {
  if (!content || typeof content !== 'string') return [];
  return content
    .split(/\r?\n/)
    .map(line => line.trim())
    .filter(line => line.length > 0 && !line.startsWith('#'))
    .map(line => {
      const parts = line.split('|').map(p => p.trim());
      const text = parts[0];
      if (!text) return null;
      const startStr = parts[1] || '';
      const endStr = parts[2] || '';
      if (!isHebahanActive(startStr, endStr, now)) return null;
      return { text, startDate: startStr, endDate: endStr };
    })
    .filter(Boolean);
}
```

> Nota: jika `parseHebahan` anda menetapkan `id` pada setiap baris (dilihat dipakai di `apiServerService`), kekalkan penetapan `id` itu di sini juga.

## 3. Backend — penjadual yang membuang item tamat tempoh sendiri

**Fail baru: `nodejs/services/hebahanScheduler.js`**

```js
/**
 * Setiap 60s: baca semula hebahan, kira set aktif, dan broadcast HANYA bila berubah.
 * Ini buang keperluan reload 12 AM untuk hebahan.
 */
function startHebahanScheduler(dataService, socketServerService, intervalMs = 60 * 1000) {
  let lastSignature = null;

  async function tick() {
    try {
      const content = await dataService.readFile('hebahan').catch(() => '');
      const active = dataService.parseHebahan(content, new Date());
      const signature = JSON.stringify(active.map(h => h.text));
      if (signature !== lastSignature) {
        lastSignature = signature;
        socketServerService.broadcastHebahanUpdate(active);
        console.log(`hebahan: set aktif berubah -> broadcast (${active.length} item)`);
      }
    } catch (err) {
      console.error('hebahanScheduler tick error:', err);
    }
  }

  tick(); // jalankan sekali serta-merta
  const timer = setInterval(tick, intervalMs);
  return () => clearInterval(timer);
}

module.exports = { startHebahanScheduler };
```

**Wayar pada permulaan server** (di tempat anda init `dataService` + `socketServerService`):

```js
const { startHebahanScheduler } = require('./services/hebahanScheduler');
// selepas socketServerService.init(io):
startHebahanScheduler(dataService, socketServerService);
```

Frontend sudah ada pendengar `hebahan:updated` yang mengemas kini state tanpa reload — jadi tiada perubahan plumbing diperlukan untuk ini.

**Fail: `react/src/components/DateTimeOverlay.jsx`** — tambah ticker 60s dan tapis sebelum papar.

```jsx

// di dalam komponen, bersama useState sedia ada:

useEffect(() => {
  const id = setInterval(() => setMinuteTick(t => t + 1), 60 * 1000);
  return () => clearInterval(id);

```

`minuteTick` dalam state menyebabkan render semula setiap minit, jadi item yang tamat tengah malam hilang dalam ≤60s walaupun backend tidak broadcast.

## 5. Buang reload 12 AM (jika hebahan satu-satunya sebab)

**Fail: `react/src/contexts/DataContext.jsx`** — dalam `checkMidnight`, ganti reload penuh dengan muat semula data sahaja:

```js
const checkMidnight = useCallback((todayStr) => {
  const lastLoad = dataLoadDateRef.current
    ?? (typeof localStorage !== 'undefined' ? localStorage.getItem(DATA_LOAD_DATE_KEY) : null)
    ?? '';
  if (lastLoad && todayStr !== lastLoad) {
    dataLoadDateRef.current = todayStr;
    try { if (typeof localStorage !== 'undefined') localStorage.setItem(DATA_LOAD_DATE_KEY, todayStr); } catch (_) {}
    loadAllData();   // refresh tanpa reload penuh; takwim ikut takwim:refresh / loadTakwimOnly
  }
}, [loadAllData]);
```

> **AWAS:** sahkan tiada komponen lain bergantung pada reload penuh 12 AM (cth. peralihan tarikh/takwim/waktu solat). Jika takwim sudah dikemas kini melalui `takwim:refresh` + `loadTakwimOnly`, langkah ini selamat. Jika anda ragu, biarkan `checkMidnight` tetapi langkah 3 & 4 sudah cukup membuat hebahan menapis sendiri.

## Ringkasan kesan

| Lapisan | Sebelum | Selepas |
|---|---|---|
| Backend parse | Tapis sekali pada masa parse | Logik dikongsi + parse semula bila perlu |
| Tamat tempoh | Beku sehingga reload/CRUD | Auto-buang dalam ≤60s |
| Tarikh | Parse UTC (rapuh) | Parse waktu tempatan |
| Reload 12 AM | Diperlukan untuk hebahan | Boleh dibuang |
