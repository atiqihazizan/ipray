Peta fail dalam new-files -> lokasi sasaran di Raspberry Pi (user ipray).
Salin/cipta fail ke path di sebelah kanan (sudo untuk etc dan boot).

home-ipray/kiosk.sh
  -> /home/ipray/kiosk.sh
  chmod +x /home/ipray/kiosk.sh

home-ipray/bin/set-primary-display.sh
  -> /home/ipray/bin/set-primary-display.sh
  chmod +x /home/ipray/bin/set-primary-display.sh

home-ipray-config/lxsession/rpd-x/desktop.conf
  -> /home/ipray/.config/lxsession/rpd-x/desktop.conf

home-ipray-config/lxsession/rpd-x/autostart
  -> /home/ipray/.config/lxsession/rpd-x/autostart

home-ipray-config/autostart/gnome-keyring-secrets.desktop
  -> /home/ipray/.config/autostart/gnome-keyring-secrets.desktop

etc-systemd-system/kiosk.service
  -> /etc/systemd/system/kiosk.service
  sudo systemctl daemon-reload && sudo systemctl enable kiosk && sudo systemctl start kiosk

etc-sudoers-d/ipray-kiosk
  -> /etc/sudoers.d/ipray-kiosk
  sudo chmod 440 /etc/sudoers.d/ipray-kiosk
  (Supaya user ipray boleh jalankan systemctl reload/restart kiosk tanpa minta password.)

etc-nginx-sites-available/ipray
  -> /etc/nginx/sites-available/ipray
  sudo ln -sf /etc/nginx/sites-available/ipray /etc/nginx/sites-enabled/ipray
  sudo nginx -t && sudo systemctl reload nginx

usr-share-plymouth-themes/ipray-welcome/*
  -> /usr/share/plymouth/themes/ipray-welcome/
  (splash.png - rujuk SPLASH-NOTE.txt dalam folder ipray-welcome)
  sudo update-initramfs -u
